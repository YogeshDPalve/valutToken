The Problem

JWT gives you rope.
VaultToken removes it.

JWT's "algorithm agility" lets developers pick any algorithm — including wrong ones. VaultToken removes the choice.

⚠ JWT

Algorithm agility is a liability

  • ✗Developer picks algorithm: HS256, RS256, ES256, none
  • ✗alg:none tokens accepted by major libraries historically
  • ✗RS256→HS256 confusion attack — real and exploited
  • ✗Payload is only Base64URL — visible to anyone
  • ✗HS256 with weak secret brute-forced offline with hashcat
  • ✗kid header injectable into DB/file lookups
  • ✗No revocation standard
✅ VaultToken

Security at the specification level

  • ✓Algorithm fixed by version: v4 = XChaCha20-Poly1305 / Ed25519
  • ✓No algorithm field — alg:none is structurally impossible
  • ✓No algorithm negotiation — confusion attack impossible
  • ✓v4.local fully encrypts payload with 256-bit AEAD
  • ✓Keys are 256-bit random — no weak key API exists
  • ✓Footer is PAE-bound — tampering fails MAC immediately
  • ✓Redis JTI blocklist + refresh token family tracking built-in
How It Works

Four steps. No surprises.

The complete VaultToken flow from first key to production.

01

Generate Keys

Run npm run keygen. 256-bit keys stored AES-encrypted in Redis. Done in 2 seconds.

02

Issue Token

POST /tokens/issue returns a fully encrypted v4.local token with claims, JTI, expiry.

03

Verify Token

POST /tokens/verify decrypts, validates claims, checks revocation — returns claims or 401.

04

Rotate & Revoke

Zero-downtime key rotation. Instant JTI revocation. Automatic refresh token reuse detection.

Features

Everything included.

🔐

PASETO v4 Only

XChaCha20-Poly1305 + Ed25519. One version, zero algorithm choices.

🔄

Zero-downtime Rotation

Rotate keys while old tokens stay valid through a configurable grace period.

🚫

Token Revocation

Redis JTI sorted set. Revoke by JTI, subject, key, or refresh family. Auto-cleanup via TTL.

🔁

Refresh Token Families

Refresh reuse = entire family revoked instantly. Both attacker and user lose access.

📌

Implicit Assertions

Bind tokens to IP + User-Agent. Stolen token from different context fails MAC.

🏢

Multi-tenant

Isolated key namespaces per tenant. Cross-tenant acceptance is cryptographically impossible.

📊

Audit Log

Every operation logged. Queryable by event, subject, time range. No secrets in logs.

⚡

Rate Limiting

Sliding-window rate limits per endpoint per client. Standard rate limit headers.

🔍

Introspection

RFC 7662 token introspection endpoint. Always returns 200 — active or inactive.

Integration

Two HTTP calls.

VaultToken is a REST API. Integrate from any language, any framework.

JavaScript
// Issue token const res = await fetch('http://localhost:3000/tokens/issue', { method: 'POST', headers: { 'Content-Type': 'application/json', 'X-Api-Key': apiKey }, body: JSON.stringify({ sub: 'user_42', aud: 'api.myapp.com', claims: { role: 'admin' }, ttl: 3600, refreshable: true, }), }); const { token, refreshToken } = await res.json(); // token → "v4.local.QAxIpVe-ECVNI..." (fully encrypted) // Verify token const { valid, sub, claims } = await fetch('http://localhost:3000/tokens/verify', { method: 'POST', headers: { 'Content-Type': 'application/json', 'X-Api-Key': apiKey }, body: JSON.stringify({ token, aud: 'api.myapp.com' }), }).then(r => r.json()); // → { valid: true, sub: "user_42", claims: { role: "admin" } }
# Issue a token curl -X POST http://localhost:3000/tokens/issue \ -H "Content-Type: application/json" \ -H "X-Api-Key: your-api-key" \ -d '{"sub":"user_42","aud":"api.myapp.com","claims":{"role":"admin"},"ttl":3600}' # Verify a token curl -X POST http://localhost:3000/tokens/verify \ -H "Content-Type: application/json" \ -H "X-Api-Key: your-api-key" \ -d '{"token":"v4.local.xxx","aud":"api.myapp.com"}' # Rotate key (admin) curl -X POST http://localhost:3000/keys/rotate \ -H "X-Admin-Key: your-admin-key" \ -H "Content-Type: application/json" \ -d '{"purpose":"local","gracePeriod":86400}'
import httpx client = httpx.Client( base_url="http://localhost:3000", headers={"X-Api-Key": "your-api-key"} ) # Issue r = client.post("/tokens/issue", json={ "sub": "user_42", "aud": "api.myapp.com", "claims": {"role": "admin"}, "ttl": 3600, }) token = r.json()["token"] # "v4.local.QAxIpVe-..." # Verify result = client.post("/tokens/verify", json={ "token": token, "aud": "api.myapp.com" }).json() # → { "valid": True, "sub": "user_42", "claims": {"role": "admin"} }
import ("bytes"; "encoding/json"; "net/http") func issueToken(apiKey string) (string, error) { body, _ := json.Marshal(map[string]any{ "sub": "user_42", "aud": "api.myapp.com", "claims": map[string]any{"role": "admin"}, "ttl": 3600, }) req, _ := http.NewRequest("POST", "http://localhost:3000/tokens/issue", bytes.NewReader(body)) req.Header.Set("Content-Type", "application/json") req.Header.Set("X-Api-Key", apiKey) resp, _ := http.DefaultClient.Do(req) var result map[string]any json.NewDecoder(resp.Body).Decode(&result) return result["token"].(string), nil }
